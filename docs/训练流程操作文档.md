每个 HDF5 文件内部结构：

## Step 1：数据格式转换

将 HDF5 数据转换为 LeRobot 格式：

```bash
cd /media/jiayueru/Ckpt/WRL/openpi
UV_INDEX_URL=https://pypi.org/simple uv run /media/jiayueru/Ckpt/WRL/openpi/examples/libero/convert_own_data_to_lerobot_abs.py \
  --data-dir /media/jiayueru/Ckpt/WRL/data/WorldRL/pick_flower \
  --repo-name pick_flower \
  --output-dir /media/jiayueru/Ckpt/WRL/data/processed_lerobot/WorldRL/pick_flower \
  --action-interval 1 \
  --overwrite \
  --task "pick up the flower and put it in the vase"
```
对于每个位置的任务，命名标准为：input:WorldRL-<统一任务名称>/position<x>_<date> output:processed_position<x>_<date>

**参数说明：**

| 参数 | 说明 |
|------|------|
| `--data-dir` | 原始 HDF5 文件所在目录（支持递归搜索一层子目录） |
| `--repo-name` | 输出数据集名称，后续 config 中 `repo_id` 必须与此一致 |
| `--output-dir` | 转换结果的父目录，实际写入到 `output-dir/repo-name/` |
| `--overwrite` | 目标目录已存在时覆盖删除，不加此参数会报错 |
| `--task` | 语言任务描述，写入每帧的 `task` 字段，训练时作为 prompt |
| `--action-interval` | delta action 帧间隔 N（默认 1），控制频率低时可改为 2 或 3 |
| `--image-size` | 输出图像边长（默认 256），训练前还会 resize 到 224 |

**转换后数据结构：**

```
processed_data/pickplace_bread_pos1/
├── data/
│   └── chunk-000/
│       ├── episode_000000.parquet   ← 每帧: image, wrist_image, state(8,), actions(7,)
│       └── ...
├── meta/
│   ├── info.json          ← 数据集元信息 (fps, total_episodes, features 等)
│   ├── episodes.jsonl     ← 每 episode 的起止帧索引
│   ├── tasks.jsonl        ← 语言任务描述
│   └── episodes_stats.jsonl
└── videos/
```

**state / action 约定：**

- `state (8D)` = `joint(7D)` + `gripper_binary(1D)`，其中 `0=open`，`1=closed`
- `action (7D)` = `delta_pose(6D)` + `gripper_binary(1D)`
  - `delta_pose[i] = pose[i+N] - pose[i]`，末尾帧 delta=0

## Step 2：注册训练配置

编辑 `src/openpi/training/config.py`，在文件末尾 `_CONFIGS` 列表中参照以下模板添加新配置（约第 788 行之后）：

```python
TrainConfig(
    name="pi05_bread_pos0",          # 唯一名称，后续所有命令用此名称引用
    model=pi0_config.Pi0Config(
        pi05=True,
        action_horizon=10,            # action chunk 长度，推理时每次预测 10 步动作
        discrete_state_input=False,
    ),
    data=LeRobotLiberoDataConfig(
        repo_id="pickplace_bread_pos0",  # 必须与 --repo-name 完全一致
        base_config=DataConfig(
            prompt_from_task=True,    # 从 tasks.jsonl 读取语言指令作为 prompt
        ),
        extra_delta_transform=False,  # 已是 delta action，无需再次转换
    ),
    batch_size=32,
    lr_schedule=_optimizer.CosineDecaySchedule(
        warmup_steps=300,
        peak_lr=5e-5,
        decay_steps=1_000_000,
        decay_lr=5e-5,
    ),
    optimizer=_optimizer.AdamW(clip_gradient_norm=1.0),
    ema_decay=0.69,
    weight_loader=weight_loaders.CheckpointWeightLoader(
        # 本地 pi0.5 预训练权重路径
        "/home/franka/WorldRLCodes/openpi/model/hairuoliu/pi05_base/params"
    ),
    num_train_steps=30_000,           # 数据量少时建议 5k~10k 步
),
```

**关键参数说明：**

| 参数 | 说明 |
|------|------|
| `name` | 配置唯一标识，后续命令的 `--config-name` 参数 |
| `repo_id` | 必须与 Step 1 中 `--repo-name` 完全一致 |
| `extra_delta_transform` | 脚本已计算 delta pose，此处保持 `False` |
| `prompt_from_task` | `True` 时从 `tasks.jsonl` 读取语言指令 |
| `action_horizon` | 推理时每次预测的动作步数，建议与训练一致（10） |
| `num_train_steps` | 20 个 episode 约 12k 帧，建议先跑 10k 步观察 loss 曲线 |
| `weight_loader` | 加载预训练权重路径，pi0.5 权重位于上方路径 |

ensure_conda_env() {
  log "阶段1: ensure_conda_env"
  "$CONDA_BIN" tos accept --override-channels --channel https://repo.anaconda.com/pkgs/main || true
  "$CONDA_BIN" tos accept --override-channels --channel https://repo.anaconda.com/pkgs/r || true

  if "$CONDA_BIN" env list | awk '{print $1}' | grep -qx "$ENV_NAME"; then
    log "环境 $ENV_NAME 已存在"
  else
    "$CONDA_BIN" create -n "$ENV_NAME" python=3.11 -c conda-forge -y

ensure_system_deps() {
  log "阶段2: ensure_system_deps（缺失才安装，apt 不走代理）"

  local missing=()
  dpkg -s cmake >/dev/null 2>&1 || missing+=("cmake")
  dpkg -s build-essential >/dev/null 2>&1 || missing+=("build-essential")

  if [ "${#missing[@]}" -eq 0 ]; then
    log "系统编译依赖已满足，跳过 apt"
    return
  fi

  local _http_proxy="${http_proxy-}" _https_proxy="${https_proxy-}" _HTTP_PROXY="${HTTP_PROXY-}" _HTTPS_PROXY="${HTTPS_PROXY-}" _ALL_PROXY="${ALL_PROXY-}"
  unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY ALL_PROXY

  apt-get update
  apt-get install -y --fix-missing "${missing[@]}"

  [ -n "${_http_proxy}" ] && export http_proxy="${_http_proxy}"
  [ -n "${_https_proxy}" ] && export https_proxy="${_https_proxy}"
  [ -n "${_HTTP_PROXY}" ] && export HTTP_PROXY="${_HTTP_PROXY}"
  [ -n "${_HTTPS_PROXY}" ] && export HTTPS_PROXY="${_HTTPS_PROXY}"
  [ -n "${_ALL_PROXY}" ] && export ALL_PROXY="${_ALL_PROXY}"

  log "系统编译依赖安装完成"
}

## 附录：常见问题

### Q1：训练时提示 `Norm stats not found`

`HF_LEROBOT_HOME` 环境变量未设置，或 Step 3 未正常执行。检查：

```bash
ls ./assets/pi05_bread_pos1/pickplace_bread_pos1/
```

### Q2：`repo_id not found` 或数据集加载失败

检查环境变量是否正确指向数据目录：

```bash
export HF_LEROBOT_HOME=/home/franka/tactile/data/WorldRL-pickplace-bread-full/processed_data
ls $HF_LEROBOT_HOME/pickplace_bread_pos1/meta/info.json
```

### Q3：显存 OOM

依次尝试：
1. 降低 batch size：`--batch-size 16` 或 `--batch-size 8`
2. 降低 XLA 显存分配：`XLA_PYTHON_CLIENT_MEM_FRACTION=0.7`
3. 使用 LoRA 微调（见下方配置示例）

### Q4：loss 不下降 / 训练不稳定

- 检查 `norm_stats.json` 是否正常生成（非空）
- 检查 gripper 状态约定：数据中 `0=open, 1=closed`（Pi0 约定）
- 可视化检查：通过 `meta/episodes_stats.jsonl` 确认 actions 范围合理

### Q5：使用 LoRA 低显存微调

在 `config.py` 中修改 model 配置：

```python
model=pi0_config.Pi0Config(
    pi05=True,
    action_horizon=10,
    discrete_state_input=False,
    paligemma_variant="gemma_2b_lora",   # 启用 LoRA
    action_expert_variant="gemma_300m_lora",
),
freeze_filter=pi0_config.Pi0Config(
    pi05=True,
    paligemma_variant="gemma_2b_lora",
    action_expert_variant="gemma_300m_lora",
).get_freeze_filter(),
ema_decay=None,  # LoRA 微调关闭 EMA
```
